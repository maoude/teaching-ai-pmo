# AI-Assisted Gamified Module Overview

## Module Title

From Plan to Control: Managing a Project Like a Real PMO

## Course Context

This module is designed for a university course in project management, software engineering management, engineering management, information systems, or business management.

## Target Learners

Third- or fourth-year undergraduate students, or first-year master's students, in engineering, computer science, business, management, or related fields.

## Duration

3 hours, or two sessions of 90 minutes.

## Main Scenario

The university wants to build a Smart Campus Mobile App in 12 weeks. The app includes login, course schedule, exam calendar, notifications, room reservation, complaint/request system, admin dashboard, and pilot deployment.

Students act as a project management office team. They define the scope, build the WBS, prepare the schedule, monitor progress, diagnose delays, and present a recovery plan after a project crisis.

## Learning Objectives

By the end of the module, students will be able to:

1. Explain the project management cycle from initiation to closing.
2. Build a deliverable-oriented Work Breakdown Structure.
3. Convert a WBS into tasks, dependencies, milestones, and responsibilities.
4. Estimate duration, cost, resources, and risks for a realistic project.
5. Optimize a project schedule under time and budget constraints.
6. Interpret project tracking indicators including PV, EV, AC, SV, CV, SPI, and CPI.
7. Diagnose project problems from dashboard data.
8. Propose corrective actions for a late, over-budget, or unstable project.
9. Communicate a professional PMO status or recovery briefing.

## AI Tools Used

| Tool | Role in the Module |
| --- | --- |
| NotebookLM | Source-grounded summaries of project management concepts from uploaded course notes or references |
| ChatGPT, Gemini, or Claude | Scenario writing, WBS examples, crisis cards, feedback prompts, and activity instructions |
| Gamma | First draft of lecture slides |
| Canva | Final visual design, worksheets, badges, diagrams, and briefing templates |
| Genially | Branching simulation and interactive decision screens |
| Google Sheets or Excel Online | Schedule, budget, earned value, and dashboard simulation |
| Quizizz, QuestionWell, or MagicSchool | Quizzes, exit ticket, and formative assessment |
| Mentimeter or Wooclap | Live polls and group decisions |

## Learning Activity Types

| Learning Type | Activity |
| --- | --- |
| Acquisition | Short instructor explanation supported by slides and visual diagrams |
| Inquiry | Students analyze the project case and identify stakeholders, risks, and constraints |
| Practice | Students calculate schedule and cost indicators from dashboard data |
| Production | Groups create a WBS, schedule, risk register, and PMO recovery briefing |
| Collaboration | Groups work as PMO teams and negotiate corrective actions |
| Discussion | Class debates project health, trade-offs, and sponsor communication |
| Reflection | Exit ticket on what makes project control effective |

## Final Student Outputs

Each group submits:

1. Project charter summary
2. WBS
3. Schedule table
4. Risk register
5. Tracking dashboard interpretation
6. Corrective action report
7. Three-minute PMO recovery briefing

## Classroom Readiness

The module can be delivered with a slide deck, a Genially branching simulation, a Google Sheets dashboard, printable or digital worksheets, and a short quiz.
