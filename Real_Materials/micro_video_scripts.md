# Micro-Video Scripts

## Video 1: What Is a WBS?

### Length

90 seconds

### Script

A Work Breakdown Structure, or WBS, is a deliverable-oriented decomposition of project work.

That means it does not simply list random tasks. It breaks the project into outputs that must be delivered.

For the Smart Campus Mobile App, the main deliverables are project management, requirements, design, development, testing, and deployment.

Each deliverable can be broken down further. For example, development may include authentication, course schedule, notifications, room reservation, and admin dashboard.

A good WBS helps the PM estimate time, assign resources, identify risks, and track progress.

If the WBS is weak, the schedule and dashboard will also be weak.

## Video 2: How to Read a Project Dashboard

### Length

2 minutes

### Script

A project dashboard helps the PM compare the plan with actual performance.

In our case, by week 6 the project should be 50 percent complete, but actual progress is only 38 percent. This means the project is behind schedule.

The planned cost is 20,000 USD, but the actual cost is 24,000 USD. This means the project is spending more than planned.

There are also 12 critical bugs, while the planned number was zero. This is a quality and technical risk.

A dashboard does not solve the problem by itself. The PM must interpret the data and decide corrective action.

The responsible conclusion is that the project is late, over budget, and technically unstable.

## Video 3: How to Recover a Failing Project

### Length

2 minutes

### Script

When a project is failing, the PM should not hide the problem. The PM should diagnose the cause, communicate honestly, and propose corrective action.

In the Smart Campus App project, the week 8 crisis includes database integration failure, many bugs, sponsor pressure, and team overload.

The best action is to freeze non-critical scope and focus on the most important deliverables: login, schedule, notifications, and a stable pilot.

Stopping testing is dangerous because it hides quality problems. Adding more features is also dangerous because it increases scope.

A professional recovery briefing explains what happened, what the impact is, what action is proposed, and what decision is needed from the sponsor.
